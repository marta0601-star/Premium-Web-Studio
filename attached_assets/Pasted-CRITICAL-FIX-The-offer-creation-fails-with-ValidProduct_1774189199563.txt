CRITICAL FIX: The offer creation fails with "ValidProductization" error. In many Allegro categories (especially food/beverages), every offer MUST be linked to a catalog product. If the product doesn't exist in the catalog, we must CREATE it first.

Fix the flow for products NOT found in Allegro catalog:

STEP 1 — CREATE PRODUCT IN ALLEGRO CATALOG:
- Use POST https://api.allegro.pl/sale/products with:
  {
    "name": "product name",
    "category": {"id": "categoryId"},
    "parameters": [array of required parameters with values],
    "images": ["image_url"],
    "ean": ["ean_code"]
  }
- Headers: Authorization: Bearer {token}, Accept: application/vnd.allegro.public.v1+json, Content-Type: application/vnd.allegro.public.v1+json
- This creates the product in Allegro catalog and returns a product.id

STEP 2 — CREATE OFFER LINKED TO THAT PRODUCT:
- Use the returned product.id in the offer creation
- Set "product": {"id": "returned_product_id"} in the offer
- This satisfies the ValidProductization requirement
